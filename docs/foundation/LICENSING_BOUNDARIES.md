## Core Principle

**Tools are external. KEYS are original structure. Licensing never contaminates core IP.**

---

## Licensing Model

### Tools Are External
- **Tools**: Cursor, Jupyter, GitHub, Stripe, Supabase, etc.
- **Ownership**: Owned by tool vendors
- **Licensing**: Governed by tool vendor licenses
- **KEYS Role**: KEYS does not license tools

### KEYS Are Original Structure
- **Keys**: Original structure, patterns, and organization
- **Ownership**: Owned by KEYS or key creators
- **Licensing**: Governed by KEYS marketplace licenses
- **KEYS Role**: KEYS licenses key structure

---

## Key Licensing

### License Types

#### Proprietary (Default)
- **Use**: Personal or organizational use
- **Modification**: Not allowed
- **Redistribution**: Not allowed
- **Commercial Use**: Allowed for purchased keys
- **Sublicensing**: Not allowed

#### MIT / Apache 2.0
- **Use**: Personal or commercial use
- **Modification**: Allowed
- **Redistribution**: Allowed with attribution
- **Commercial Use**: Allowed
- **Sublicensing**: Allowed

#### GPL / AGPL
- **Use**: Personal or commercial use
- **Modification**: Allowed
- **Redistribution**: Required (copyleft)
- **Commercial Use**: Allowed
- **Sublicensing**: Required (copyleft)

#### Creative Commons
- **Use**: Personal or commercial use (varies by variant)
- **Modification**: Allowed (varies by variant)
- **Redistribution**: Allowed (varies by variant)
- **Commercial Use**: Allowed (varies by variant)
- **Sublicensing**: Allowed (varies by variant)

### License Assignment
- **Key Creator**: Assigns license when creating key
- **KEYS Platform**: Enforces license terms
- **Users**: Must comply with license terms
- **Violations**: Handled according to license terms

- **Cursor**: Proprietary (Cursor license)

- **GitHub**: Proprietary (GitHub license)
- **Stripe**: Proprietary (Stripe license)
- **Supabase**: Apache 2.0 (Supabase license)

### KEYS Compliance
- **KEYS**: Complies with all tool vendor licenses
- **Users**: Must comply with tool vendor licenses
- **KEYS Role**: KEYS does not modify tool licenses

## IP Boundaries

### KEYS IP
- **Key Structure**: Original structure and organization
- **Patterns**: Original patterns and workflows
- **Documentation**: Original documentation
- **Metadata**: Original metadata and taxonomy

### Tool IP
- **Tool Code**: Owned by tool vendors
- **Tool APIs**: Owned by tool vendors
- **Tool Documentation**: Owned by tool vendors
- **Tool Branding**: Owned by tool vendors

### Clear Separation
- **KEYS**: Never claims ownership of tool IP
- **Tools**: Never claim ownership of KEYS IP
- **Users**: Understand IP boundaries

---

## Marketplace Licensing

### Marketplace Terms
- **Key Purchase**: Grants license to use key
- **Key Updates**: Includes updates for license period
- **Key Redistribution**: Prohibited (unless license allows)
- **Key Modification**: Prohibited (unless license allows)

### User Rights
- **Use**: Right to use purchased keys
- **Updates**: Right to receive updates during license period
- **Support**: Right to access support and documentation
- **Refunds**: Right to refunds per refund policy

### User Obligations
- **License Compliance**: Must comply with key licenses
- **Tool Compliance**: Must comply with tool licenses
- **No Redistribution**: Cannot redistribute keys (unless license allows)
- **No Modification**: Cannot modify keys (unless license allows)

---

## Contributor Licensing

### Contributor Rights
- **Ownership**: Contributors own their contributions
- **Licensing**: Contributors assign licenses to contributions
- **Attribution**: Contributors receive attribution
- **Revenue**: Contributors receive revenue share (if applicable)

### Contributor Obligations
- **Original Work**: Contributions must be original work
- **License Assignment**: Contributors must assign licenses
- **Tool Compliance**: Contributions must comply with tool licenses
- **KEYS Compliance**: Contributions must comply with KEYS policies

### Contributor Agreements
- **License Grant**: Contributors grant licenses to KEYS
- **Warranty**: Contributors warrant originality
- **Indemnification**: Contributors indemnify KEYS
- **Termination**: KEYS can terminate contributor agreements

## Legal Defensibility

### Why This Prevents Legal Dead-Ends

#### Clear Boundaries
- **KEYS IP**: Clearly defined
- **Tool IP**: Clearly defined
- **User Rights**: Clearly defined
- **No Ambiguity**: No legal ambiguity

#### License Compliance
- **Tool Licenses**: KEYS complies with all tool licenses
- **Key Licenses**: Users comply with key licenses
- **No Violations**: No license violations

#### IP Protection
- **KEYS IP**: Protected by licensing
- **Tool IP**: Protected by tool vendor licenses
- **User IP**: Protected by user licenses
- **No Contamination**: No IP contamination

### Risk Mitigation

#### Tool Vendor Risk
- **Compliance**: KEYS complies with tool licenses
- **Attribution**: KEYS attributes tool vendors
- **No Competition**: KEYS doesn't compete with tools
- **Risk**: Low

#### User Risk
- **Clear Terms**: Clear licensing terms
- **Compliance**: Users can comply easily
- **Support**: KEYS provides support
- **Risk**: Low

#### Contributor Risk
- **Clear Agreements**: Clear contributor agreements
- **Originality**: Contributors warrant originality
- **Indemnification**: Contributors indemnify KEYS
- **Risk**: Low

## License Compatibility

### Compatible Licenses
- **MIT + MIT**: Compatible
- **Apache 2.0 + Apache 2.0**: Compatible
- **GPL + GPL**: Compatible
- **Proprietary + Proprietary**: Compatible

### Incompatible Licenses
- **GPL + Proprietary**: Incompatible
- **AGPL + Proprietary**: Incompatible
- **Copyleft + Proprietary**: Incompatible

### Bundle Licensing
- **Same License**: All keys have same license
- **Compatible Licenses**: All keys have compatible licenses
- **License Disclosure**: Bundle license disclosed clearly
