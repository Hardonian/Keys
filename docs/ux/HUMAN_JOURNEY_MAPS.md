# Human-Centered Journey Maps

This document validates the customer journey for KEYS across different user personas, identifying anxiety spikes and how the UX addresses them.

## Journey Structure

Each journey follows this path:
1. **First Visit** → First impression, initial understanding
2. **First Click** → Entry point selection, initial engagement
3. **Discovery** → Finding relevant Keys
4. **Purchase** → Adding Keys to Keyring
5. **Usage** → Using Keys in practice

---

## Persona 1: Non-Technical Founder

### Profile
- Technical comfort: Low
- Primary concern: "Am I behind? Do I need to learn coding?"
- Goal: Ship products without technical debt

### Journey Map

#### First Visit
**Emotional State:** Anxious, uncertain
**Anxiety Spikes:**
- ❌ "This looks like it's for developers only"
- ❌ "I don't understand the technical jargon"

**UX Solutions:**
- ✅ Hero message: "You're not behind. You already have the tools."
- ✅ Situation-based entry tiles use human language, not tech terms
- ✅ No skill-level labels (no "beginner/advanced")
- ✅ Clear explanation: KEYS ≠ framework, ≠ AI autopilot

**Validation:** Non-technical user can understand value proposition without reading docs

#### First Click
**Emotional State:** Cautiously optimistic
**Anxiety Spikes:**
- ❌ "Will I understand what I'm clicking into?"
- ❌ "Am I making a mistake?"

**UX Solutions:**
- ✅ Situation tiles clearly describe outcomes ("I'm responsible for something breaking")
- ✅ Each tile explains what it prevents, not what it requires
- ✅ Pre-filtered discovery flow reduces cognitive load

**Validation:** User clicks without hesitation, understands where they're going

#### Discovery
**Emotional State:** Engaged, exploring
**Anxiety Spikes:**
- ❌ "How do I know which Key is right?"
- ❌ "What if I pick the wrong one?"

**UX Solutions:**
- ✅ Guided discovery flow asks about situation, not technical skills
- ✅ Recommendations include "Why this Key" explanations
- ✅ Clear opt-out without penalty
- ✅ Keys presented as situations, not products

**Validation:** User feels guided, not tested

#### Purchase
**Emotional State:** Decisive, prepared
**Anxiety Spikes:**
- ❌ "What am I actually buying?"
- ❌ "Will this run code automatically?"
- ❌ "What if I don't use it right away?"

**UX Solutions:**
- ✅ Pre-checkout summary: "What unlocks" + "What does NOT happen"
- ✅ Clear: No code runs, no creds stored, no changes to tools
- ✅ Ownership messaging: "Permanent access, yours forever"
- ✅ Language: "Add to Keyring" (preparedness) not "Buy now" (urgency)

**Validation:** Purchase feels like preparedness, not commitment

#### Usage
**Emotional State:** Confident, supported
**Anxiety Spikes:**
- ❌ "I haven't used this in a while, did I waste money?"
- ❌ "Is this still relevant?"

**UX Solutions:**
- ✅ Value reminder: "This still applies"
- ✅ Version visibility shows currency
- ✅ Keyring groups by situation, shows coverage
- ✅ No "You haven't used this" messaging

**Validation:** User feels prepared, not pressured

- ❌ "Will my team be able to use this?"

- ✅ Situation-based routing to relevant Keys

#### Discovery
**Emotional State:** Analytical, comparing
**Anxiety Spikes:**

- ❌ "What's the difference between Keys?"

- ✅ Recommendations explain "Why this Key"

**Anxiety Spikes:**
- ❌ "Is this a one-time cost or subscription?"

#### Usage

**Anxiety Spikes:**

**Validation:** User feels prepared, sees coverage clearly

## Common Anxiety Patterns & Solutions

### Pattern 1: "I don't know where to start"
**Solution:** Situation-based entry tiles, guided discovery flow

### Pattern 2: "What am I actually buying?"
**Solution:** Pre-checkout summary, clear ownership messaging

### Pattern 3: "Will this waste my time?"
**Solution:** 4 parallel views, clear scope boundaries, preview options

### Pattern 4: "Am I behind?"
**Solution:** Hero messaging normalizes uncertainty, emphasizes existing tools

### Pattern 5: "What if I don't use it?"
**Solution:** Value reminders, permanent access messaging, no engagement pressure

## Validation Criteria

### For Non-Technical Users
✅ "This makes me feel calmer and more capable"

### For Technical Users
✅ "This respects my intelligence and time"

### For All Users
✅ No anxiety spikes during journey
✅ Clear understanding at each step
✅ Feels like preparedness, not pressure
✅ Can explore without commitment
✅ Ownership feels permanent and valuable

## Journey Metrics

### Success Indicators
- Low bounce rate on first visit
- High engagement with situation tiles
- Completion of discovery flow
- Purchase completion rate
- Return usage of purchased Keys

### Anxiety Reduction Indicators
- Time to first click < 30 seconds
- Discovery flow completion > 60%
- Purchase abandonment < 20%
- Keyring engagement > 40% of purchasers
